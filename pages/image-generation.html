AI Image Generation

Image Genaration In Local

Rethinking local image generation. Prompt-driven, open-source, and completely offline. Focus on creating stunning images, not wrestling with technical parameters. Built on Stable Diffusion XL with intelligent automation.

Python 3.10+ PyTorch 2.1 Gradio UI SDXL Architecture GPT-2 Expansion CUDA 12.1

The Philosophy

Designed as an intelligent wrapper around Stable Diffusion XL, this tool automates the complex "tweak-test-repeat" loop typical of traditional SD WebUIs. No more endless parameter adjustments.

By integrating an offline GPT-2 prompt expansion engine, simple inputs like "house in garden" transform into elaborate, style-consistent prompts dynamically. It employs a custom advanced k-diffusion sampler with native refiner swapping, ensuring superior quality without manual tuning.

Zero Configuration: Users only need prompts - no parameter expertise required. Completely Offline: All processing happens locally on your machine. Professional Quality: Built-in optimizations deliver stunning results automatically.

Why This Tool?

Traditional image generation tools overwhelm users with hundreds of technical parameters. This platform strips away the complexity while maintaining professional-grade output quality.

Built on the powerful SDXL (Stable Diffusion XL) architecture with native 1024x1024 resolution, dual text encoders (CLIP + OpenCLIP), and superior composition understanding. Enhanced with custom optimizations for sharpness, color accuracy, and coherent details.

Inspired by the best aspects of Midjourney's simplicity, Stable Diffusion's flexibility, and ComfyUI's efficiency - combined into a single streamlined experience.

Core System Components

◆ Entry Point: entry_with_update.py

Application entry with auto-update capability using pygit2 for Git operations. Automatically fetches updates from origin remote, performs fast-forward merge if available, and falls back gracefully on update failures before launching the main application.

◆ Environment Setup: launch.py

Prepares the environment and downloads required models. Installs PyTorch, xformers, and dependencies. Downloads VAE approximation models (xlvaeapp.pth, vaeapp_sd15.pth), prompt expansion model (335 MB), and default SDXL checkpoints based on selected preset (Default, Realistic, or Anime).

◆ Web Interface: webui.py (1129 lines)

Gradio-based user interface featuring prompt input fields (positive/negative), style selection system, image input tabs (Upscale, Variation, Inpaint, Outpaint), advanced settings (sampler, scheduler, CFG, steps), and ControlNet/IP-Adapter image prompts. Clean, intuitive design for both beginners and advanced users.

◆ Async Worker: async_worker.py (1486 lines)

Main image generation pipeline handling the async processing loop. Key functions include process_task() for single image generation, process_prompt() for applying styles and prompt expansion, apply_control_nets() for IP-Adapter and ControlNet, apply_vary() for image variations, apply_inpaint() for inpainting with masks, and apply_upscale() for image upscaling.

◆ Default Pipeline: default_pipeline.py

Model management and diffusion process coordinator. Functions include refresh_base_model() to load/refresh base SDXL model, refresh_refiner_model() for refiner loading, refresh_loras() to apply LoRA weights, clip_encode() for encoding prompts via CLIP, and process_diffusion() as the main diffusion sampling orchestrator.

◆ Core Module: core.py

Low-level model operations including StableDiffusionModel class as main model container (unet, vae, clip), VAEApprox for fast VAE approximation during previews, load_model() for loading checkpoint files, ksampler() as K-diffusion sampling wrapper, and decode_vae()/encode_vae() for latent ↔ image conversion.

◆ Patch Module: patch.py

Custom enhancements to the diffusion process including Adaptive Anisotropic Filtering for improved sharpness, Negative ADM Guidance for better CFG contrast at high resolution, Adaptive CFG for dynamic guidance scale adjustment, and ControlNet Softness for smoother control signal application.

Why SDXL? Native 1024x1024 resolution • Dual text encoder (CLIP + OpenCLIP) • Superior composition and detail • Refined aesthetic understanding

◆ Text Encoders (CLIP)

Dual encoder architecture: CLIP ViT-L/14 (OpenAI's text encoder) and OpenCLIP ViT-G/14 (larger open-source variant). Process: Prompt → Tokenization → Text Embeddings → Cross-attention in UNet.

Configuration: Clip Skip set to 2 (default) - skips last 2 CLIP layers for better stylization control and artistic flexibility.

◆ GPT-2 Prompt Expansion Engine

Location: models/prompt_expansion/ containing pytorch_model.bin (~335 MB GPT-2 weights), tokenizer.json (BPE tokenizer), and positive.txt (prompt templates).

How It Works: Takes user's short prompt → GPT-2 generates expanded, detailed prompt → Adds quality-enhancing terms automatically. Similar to Midjourney's hidden prompt processing magic.

Example Transformation:
Input: "a cat"
Output: "a cat, highly detailed fur, soft lighting, professional photography, sharp focus, 8k resolution, masterpiece"

◆ VAE (Variational Autoencoder)

Encode/decode between pixel space and latent space. Models include SDXL VAE (main encoder/decoder) and VAE Approx (xlvaeapp.pth) for fast preview during generation.

Process: Image (1024×1024×3) → VAE Encode → Latent (128×128×4) → Diffusion Process → Latent (128×128×4) → VAE Decode → Image (1024×1024×3)

◆ Inpaint Models

Version: inpaint_v26.patch (1.28 GB) - Custom inpainting head for natural fill-in with better edge blending, more coherent content generation, and enhanced respect for surrounding context compared to standard SDXL inpaint.

◆ ControlNet & IP-Adapter

Supported control types: ImagePrompt (IP-Adapter for style/content transfer), FaceSwap (InsightFace for face replacement), PyraCanny (ControlNet Canny for edge-guided generation), and CPDS (ControlNet CPDS for depth/structure control).

◆ LoRA (Low-Rank Adaptation)

Advanced Features & Optimizations

◆ Prompt Magic (V2 Style)

GPT-2 model trained on high-quality prompts automatically expands simple inputs and adds quality-enhancing keywords. Transforms basic descriptions into professional-grade prompts that produce stunning results without manual optimization.

◆ Native Refiner Swap

Unlike other implementations, refiner swap occurs within a single k-sampler run, preserving sampler momentum, ODE history parameters, and sampling continuity. This results in smoother transitions and higher quality final images.

◆ Negative ADM Guidance

Compensates for lack of CFG contrast at the highest resolution level of SDXL. Negative prompts use 0.8× scale while positive prompts use 1.5× scale for optimal guidance.

if prompt_type == "negative":
  width *= 0.8; height *= 0.8
elif prompt_type == "positive":
  width *= 1.5; height *= 1.5

◆ Self-Attention Guidance (Sharpness)

Prevents overly smooth or plastic appearance using adaptive anisotropic filtering. Applies dynamic sharpness correction based on global diffusion progress (alpha = 0.001 × sharpness × progress).

◆ TSNR Correction

When CFG > 10, applies dynamic correction to prevent over-saturation. Uses adaptive CFG blending where mimicked_cfg = 7.0 (default) and smoothly interpolates based on timestep for balanced guidance without color explosion.

◆ Memory Optimization Techniques

Model Offloading: Move unused models to CPU automatically • Attention Optimization: xformers or PyTorch SDPA for efficient attention • VAE Tiling: Process large images in tiles • FP16/BF16: Half-precision computation • Gradient Checkpointing: Trade compute for memory
